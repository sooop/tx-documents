## 워크플로우

1. 스테이지된 파일 목록을 확인합니다. 스테이지된 파일이 없으면 프로젝트 루트에서 `git add .`을 사용하여 모든 변경 사항을 스테이지로 올립니다.
2. 스테이지된 파일 중에서 로그 파일이나 임시파일은 `git restore --staged` 명령을 사용하여 제외합니다.
3. 커밋할 파일의 내용을 분석하여 커밋메시지를 작성합니다.
4. 커밋합니다.

### 커밋 메시지 작성 규칙

1. `[개선]`, `[수정]`, `[기능]` 등 가장 주요한 변경에 대해 말머리를 달고 제목을 작성합니다.
2. 한 줄을 띄웁니다.
3. 제목 아래에는 주요 변경점을 최대 4개까지 목록으로 작성합니다.
