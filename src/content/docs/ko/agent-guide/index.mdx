Agent를 활용하면 원본 문서를 빠르게 가이드 페이지로 변환할 수 있습니다.
아래 절차를 따라 작업합니다.

## 작성 절차

### 1단계: 이미지 준비

필요한 이미지가 있다면 `public/temp-image` 폴더에 복사해 둡니다.

```
public/temp-image/
  └── 작업에 사용할 이미지 파일들
```

<Callout type="info" title="임시 이미지 폴더">
  `public/temp-image`는 작업 중 임시 보관 목적으로 사용합니다.
  작성이 완료되면 이미지를 정식 경로(`public/images/{섹션}/`)로 이동하세요.
</Callout>

### 2단계: 프롬프트 구성

Agent에게 전달할 프롬프트에 다음 정보를 포함합니다:

| 항목 | 설명 |
|------|------|
| **제목** | 작성할 페이지의 제목 |
| **경로** | nav 메뉴에 있다면 해당 메뉴를 클릭해서 경로를 복사 |
| **원본 경로** | `https://smartship.oopy.io`에서 찾은 원본 문서 경로 |

### 3단계: 페이지 확인

작성이 완료되면 개발 서버에서 해당 페이지가 올바르게 표시되는지 확인합니다.

```bash
npm run dev
```

## 이미지 어노테이션 방법

스크린샷에 번호, 박스, 화살표 등의 시각적 표시를 추가하려면 annotator 도구를 사용합니다.

### 실행 방법

`tools/annotator` 경로에서 다음 명령어를 실행합니다:

```bash
npm run dev -- --open
```

별도의 개발 서버로 실행되며, annotator 페이지가 자동으로 브라우저에 열립니다.

### 사용 절차

1. 이미지 이름으로 조회하고 선택합니다.
2. 캔버스에서 어노테이션을 추가합니다 — 박스, 마커, 화살표 중 선택.
3. 우측 패널에서 각 어노테이션의 속성(번호, 색상, 레이블 등)을 설정합니다.
4. 저장하면 `src/assets/images/{섹션}/{이미지이름}-{언어}.annotations.json` 파일이 생성됩니다.

<Callout type="tip" title="어노테이션 좌표">
  모든 좌표는 이미지 크기 대비 퍼센트(0~100)로 저장됩니다.
  이미지 해상도가 달라져도 위치가 유지됩니다.
</Callout>

어노테이션 JSON 형식과 세부 사항은 [콘텐츠 작성 가이드](/ko/authoring-guide/)의 "어노테이션 JSON 작성법" 섹션을 참고하세요.
