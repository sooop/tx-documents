API 연동 시 주문 수집·확인·출하지시·재고 동기화 등 동작을 세부적으로 조절할 수 있는 옵션을 제공합니다. 옵션은 **공통 옵션**과 **채널별 추가 옵션**으로 나뉩니다.

## 공통 옵션

<AnnotatedImage src="channel-api-common" section="api-guide" alt="쇼핑몰 주문 처리 설정 공통 옵션 화면" caption="쇼핑몰 주문 처리 설정 – 공통 옵션" />

### 1. 자동 주문 연동

기본값: **사용함**

API를 통해 주기적으로 주문을 자동 수집합니다.

| 설정 | 동작 |
|------|------|
| **사용함** | 1시간에 1회 자동 실행. 최초 실행 시 당일 00시부터의 주문 수집 (발송처리·취소 주문 제외). 00시~04시(KST)에 전날 미처리 주문 추가 연동 |
| **사용안함** | 판매자가 직접 주문 연동 버튼 클릭. 쇼핑몰 주문 연동 메뉴에서 특정 날짜 주문 수동 수집 가능 |

<Callout type="info">주문량이 많을 경우 다음 주기에 처리됩니다. 네트워크 장애 시에는 수동 실행도 가능합니다.</Callout>

### 2. 자동 주문 확인

기본값: **사용함**

이커머스 판매채널에서 판매자가 주문을 확인하여 '상품 준비 중' 상태로 변경하는 프로세스입니다.

| 설정 | 동작 |
|------|------|
| **사용함** | 주문 연동과 동시에 채널의 주문을 확인 상태로 변경. 1시간에 1회 자동 실행 |
| **사용안함** | 배송 관리 메뉴에서 판매자가 수동으로 처리 |

<Callout type="info">스마트십에서 주문을 삭제해도 채널의 주문 확인 상태는 유지됩니다.</Callout>

### 3. 자동 풀필먼트 출하지시

기본값: **사용함** *(풀필먼트 서비스 이용 시에만 해당)*

| 설정 | 동작 |
|------|------|
| **사용함** | 풀필먼트 주문이 Order Confirmed 상태로 변경되는 시점에 자동으로 출하지시. 자동 주문 확인 옵션과 독립적으로 동작. 1시간에 1회 자동 실행 |
| **사용안함** | 주문 관리 메뉴에서 판매자가 수동으로 출하지시 버튼 클릭 |

### 4. 자동 재고 동기화 설정

기본값: **사용안함** *(풀필먼트 서비스 이용 시에만 해당)*

실시간 기능이 아니며, 실제 가용재고의 일부분만 판매 사이트에 표시하여 과다 판매를 방지하는 보조 기능입니다.

**지원 채널:** 티몬, 쇼피파이, 쇼피, 라자다, 쿠팡, 지마켓/옥션, 11번가, 롯데온, 틱톡, 크로스몰, 야후쇼핑, 위메프, Ecwid, Ebut

| 설정 | 동작 |
|------|------|
| **사용함** | 가용재고 비율 설정 필요. 재고 변동 시 채널의 판매 가능 수량 자동 업데이트. 스마트십 SKU와 채널 상품 연결 필수 |
| **사용안함** | 판매자가 각 채널마다 수동으로 재고 업데이트 |

## 채널별 추가 옵션

### Qoo10 JP

<AnnotatedImage src="channel-api-qoo10" section="api-guide" alt="Qoo10 JP 주문 처리 설정 화면" caption="Qoo10 JP 쇼핑몰 주문 처리 설정" />

<Callout type="warning" title="Qoo10JP 도착보장 서비스 (韓ダッシュ) 이용 중인 경우">
다음 옵션을 **사용함**으로 유지해야 서비스가 정상 제공됩니다. 변경 시 도착보장 서비스가 제공되지 않을 수 있습니다.

**자동 주문 연동 | 자동 주문 확인 | 자동 풀필먼트 출하지시 | 부분 풀필먼트**
</Callout>

**예상 배송 날짜 업데이트** *(기본값: 사용함)*

Qoo10의 QSM 및 J-QSM에 발송예정일을 자동으로 입력합니다. 주문 확인 처리 시점에 입력되며 이후 재수정은 불가합니다.

**배송 날짜 설정**

D+0부터 D+3 사이에서 선택할 수 있습니다 (토·일 제외).

**옵션이 여러 개인 주문의 상품을 분리**

상품옵션은 1개이나 실제 발송해야하는 상품이 여러 개 인 경우, 판매자 정의코드를 '콤마(,)'로 구분하여 입력해두고, 이 옵션을 "판매자 옵션코드 기준으로 분리"에 설정합니다. 주문 연동 시, 판매자 옵션코드에 ","를 기준으로 분리하여, 여러 SKU를 맵핑할 수 있게 됩니다.

### 라쿠텐 (Rakuten)

<AnnotatedImage src="channel-api-rakuten" section="api-guide" alt="라쿠텐 주문 처리 설정 화면" caption="라쿠텐 쇼핑몰 주문 처리 설정" />

**API Key 만료일**

라쿠텐 API Key는 **60일** 유효 기간을 가집니다. 만료 시 스마트십과 연동이 불가능해지므로 사전에 갱신해야 합니다.

<Callout type="warning">갱신 시 serviceSecret Key와 License Key도 함께 수정해야 합니다.</Callout>

**라쿠텐 옵션코드를 풀필먼트 주문에 사용하기**

라쿠텐 주문을 풀필먼트 주문으로 연동할 경우 **Yes**로 설정합니다.

### 쇼피파이 (Shopify)

**구매자에게 알림**

주문 상태 변화 시 구매자에게 이메일을 자동으로 발송할지 설정합니다. 발송 주체는 쇼피파이이며, 발송처리 시점부터 구매자에게 안내가 전송됩니다.
