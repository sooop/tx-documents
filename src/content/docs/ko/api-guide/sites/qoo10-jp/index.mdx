## 사전 준비 — 접속 토큰과 사용자 인증

Qoo10 JP는 별도의 API 키 발급 없이 **판매자 로그인 계정**으로 직접 접속 토큰을 발급받는 방식을 사용합니다. 스마트십은 판매자의 아이디나 비밀번호를 저장하지 않습니다.

<Callout type="warning">
Qoo10 JP 계정의 **비밀번호를 변경하면 기존 접속 토큰이 자동으로 만료**됩니다. 비밀번호 변경 후에는 반드시 스마트십에서 토큰을 갱신해야 합니다.
</Callout>

## 스마트십에서 API 연결 추가

### 신규 API 연결 추가

메뉴 **주문 > 쇼핑몰 연동 > 쇼핑몰 API 설정**으로 이동합니다.

1. 상단의 **[신규등록]** 버튼을 클릭합니다.
2. 쇼핑사이트 선택 목록에서 **Qoo10 JP** 또는 **eBay Japan**을 선택합니다.

### 연결 정보 입력 및 인증

<AnnotatedImage src="qoo10-jp-api-connect-form" section="api-guide" alt="스마트십 Qoo10 JP API 연결 정보 입력 화면" caption="JQSM 아이디 입력 후 인증받기 클릭" />

1. **쇼핑몰 샵 ID** 란에 JQSM(Qoo10 JP 판매자 어드민) 로그인 아이디를 입력합니다.
2. 하단의 **[인증받기]** 버튼을 클릭합니다.
3. `api.qoo10.jp` 팝업창이 열리면 JQSM 로그인 아이디와 비밀번호를 입력한 후 **[Generate]** 버튼을 클릭합니다.
4. 팝업창이 닫히면 인증 정보가 자동으로 저장됩니다.

## API 설정 및 연동 주문

API 연동 동작(자동 주문 수집, 자동 주문 확인, 풀필먼트 출하지시 등)을 조절하는 옵션은 **[쇼핑몰 API 설정 > 주문 처리 설정]** 에서 변경할 수 있습니다.

<Callout type="tip">
옵션별 상세 설명은 [API 연동 시 주문 처리 옵션 가이드](/ko/api-guide/order-options/)를 참고하세요. Qoo10 JP의 채널별 추가 옵션(예상 배송 날짜 업데이트, 도착보장 서비스 관련 설정 등)도 해당 페이지에서 확인할 수 있습니다.
</Callout>

### 수집 가능한 주문 상태

Qoo10 JP에서는 **新規注文(신규주문)** 및 **配送準備(배송준비)** 상태의 주문이 수집됩니다.

## 풀필먼트 상품 연결

풀필먼트 서비스 이용 시, 판매사이트 상품과 스마트십 SKU를 연결해야 센터에서 정확한 상품을 출고할 수 있습니다.

### 판매 사이트 상품에 SKU no. 등록

J-QSM(Qoo10 JP 판매자 어드민) **商品管理(상품관리) > 商品の照会/修正(상품조회/수정)** 에서 상품별로 코드를 입력합니다.

| 상품 유형 | 입력 항목 | 입력 값 |
|-----------|-----------|---------|
| 단일 상품 (옵션 없음) | 販売者商品コード (판매자상품코드) | 스마트십 SKU no. 또는 판매자 정의코드 |
| 선택 상품 (옵션 있음) | 販売者オプションコード (판매자옵션코드) | 각 옵션의 판매자 정의코드 |

<Callout type="info">
판매채널 상품과 SKU를 연결하는 자세한 방법은 [API 연결 후 판매사이트 설정 방법](/ko/api-guide/site-settings/)을 참고하세요.
</Callout>

## API 키 관리

### 토큰 갱신 방법

Qoo10 JP 계정 비밀번호를 변경했거나 인증이 만료된 경우, 다음과 같이 토큰을 갱신합니다.

1. 메뉴 **주문 > 쇼핑몰 연동 > 쇼핑몰 API 설정**으로 이동합니다.
2. 갱신할 Qoo10 JP 샵을 선택합니다.
3. 인증 정보 란의 **[토큰 갱신하기]** 버튼을 클릭합니다.
4. Qoo10 JP 인증 창에서 아이디와 비밀번호를 입력합니다.
5. 인증이 완료되면 토큰이 자동으로 갱신됩니다.
6. **[연결 테스트하기]** 버튼으로 정상 작동 여부를 확인합니다.

<Callout type="warning">
토큰이 만료된 기간 동안 수집되지 않은 주문은 아래 방법으로 수동으로 가져와야 합니다.
</Callout>
