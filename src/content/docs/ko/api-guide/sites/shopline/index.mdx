## 사전 준비 — ShopLine API 키 발급

ShopLine API 연결을 위해서는 다음 정보가 필요합니다. 스마트십에서 API 연결을 설정하기 전에 ShopLine 어드민에서 미리 확인 및 발급해주세요.

| 항목 | 설명 |
|------|------|
| **샵 ID** | ShopLine Admin에 로그인하는 판매자 ID |
| **Access Token** | ShopLine 어드민에서 Staff Member를 추가한 후 API Auth를 통해 발급받는 고유 키 |

### API 키 발급 절차

<AnnotatedImage src="shopline-admin-permission-security" section="api-guide" alt="ShopLine 어드민 Permission & Security 메뉴" caption="ShopLine 어드민 좌측 하단 Permission & Security 메뉴" />

1. ShopLine 어드민에 로그인한 후, 좌측 하단의 **Permission & Security** 메뉴로 이동합니다.

<AnnotatedImage src="shopline-staff-member-api-auth" section="api-guide" alt="ShopLine Staff Member 추가 - API Auth 탭" caption="Staff Member 추가 화면에서 API Auth 탭 선택 후 Give Full Access 체크" />

2. 스크롤을 내려 **Staff Member** 를 추가합니다.
3. Staff Member 추가 화면에서 **API Auth** 탭을 클릭합니다.
4. **Give Full Access** 를 체크합니다.

<AnnotatedImage src="shopline-generate-token" section="api-guide" alt="ShopLine API Token Generate 버튼" caption="[Generate] 버튼 클릭 시 API Token 생성" />

5. 스크롤을 더 내려 **[Generate]** 버튼을 누르면 API Token이 생성됩니다.

## 스마트십에서 API 연결 추가

### 신규 API 연결 추가

메뉴 **주문 > 쇼핑몰 연동 > 쇼핑몰 API 설정**으로 이동합니다.

<AnnotatedImage src="shopline-new-register-button" section="api-guide" alt="스마트십 쇼핑몰 API 설정 화면의 신규등록 버튼" caption="쇼핑몰 API 설정 화면 상단의 [신규등록] 버튼" />

1. 상단의 **[신규등록]** 버튼을 클릭합니다.

<AnnotatedImage src="shopline-select-modal" section="api-guide" alt="연결 쇼핑사이트 선택 목록의 ShopLine" caption="쇼핑사이트 선택 목록에서 ShopLine 선택" />

2. 연결할 쇼핑사이트 선택 목록에서 **ShopLine** 을 선택합니다. 이름의 일부만 타이핑하여 검색할 수 있습니다.

### 연결 정보 입력

<AnnotatedImage src="shopline-api-connect-form" section="api-guide" alt="스마트십 ShopLine API 연결 정보 입력 화면" caption="쇼핑몰 샵 ID와 Access Token 입력" />

| 항목 | 입력 내용 |
|------|-----------|
| **쇼핑몰 샵 ID** | ShopLine Admin 로그인 판매자 ID |
| **Access Token** | ShopLine 어드민에서 발급받은 Access Token |

<Callout type="tip">
`*` 표시가 있는 필드는 필수 입력 항목입니다.
</Callout>

## API 설정 및 연동 주문

API 연동 동작(자동 주문 수집, 자동 주문 확인, 풀필먼트 출하지시 등)을 조절하는 옵션은 **[쇼핑몰 API 설정 > 주문 처리 설정]** 에서 변경할 수 있습니다.

<Callout type="tip">
옵션별 상세 설명은 [API 연동 시 주문 처리 옵션 가이드](/ko/api-guide/order-options/)를 참고하세요.
</Callout>

### 수집 가능한 주문 상태

ShopLine에서는 아래 조건을 **모두 만족**하는 주문이 수집됩니다.

| 항목 | 조건 |
|------|------|
| **결제 상태** | `Paid` |
| **배송 상태** | `UnFulfilled` |

<Callout type="info">
편의점 픽업 주문으로 확인되는 경우에는 미결제 건도 수집될 수 있습니다.
</Callout>

### 판매 사이트 상품에 SKU no. 등록

ShopLine Admin **Products & Categories > Products** 메뉴에서 상품별로 **Product SKU** 를 입력합니다.

<AnnotatedImage src="shopline-product-sku-single" section="api-guide" alt="ShopLine 단일 상품 Product SKU 입력 화면" caption="옵션이 없는 단일 상품 - Quantity & Pricing 탭의 Product SKU" />

<AnnotatedImage src="shopline-product-sku-variants" section="api-guide" alt="ShopLine 선택형 상품 Variants 탭 Product SKU 입력" caption="옵션이 있는 선택형 상품 - Variants 탭의 옵션별 Product SKU" />

| 상품 유형 | 입력 항목 | 입력 값 |
|-----------|-----------|---------|
| 단일 상품 (옵션 없음) | Quantity & Pricing 탭 > Product SKU | 스마트십 SKU no. 또는 판매자 정의코드 |
| 선택 상품 (옵션 있음) | Variants 탭 > 옵션별 Product SKU | 각 옵션의 SKU no. 또는 판매자 정의코드 |

<Callout type="info">
판매채널 상품과 SKU를 연결하는 자세한 방법은 [API 연결 후 판매사이트 설정 방법](/ko/api-guide/site-settings/)을 참고하세요.
</Callout>

## API 키 관리

ShopLine에서 발급받은 API 키에는 만료일자가 있을 수 있습니다. 키가 만료되면 API 호출 권한이 사라져 주문 수집 및 송장 등록이 중단됩니다.

- API 키의 만료일자를 별도로 관리해주세요.
- 만료 전에 ShopLine 어드민에서 API 키를 재발급 받아야 합니다.
- 재발급 후에는 스마트십의 **주문 > 쇼핑몰 연동 > 쇼핑몰 API 설정** 에서 Access Token을 수정해주세요. 일반적으로 신규 키 발급 시 기존 키는 즉시 만료됩니다.

<Callout type="warning">
API 키가 만료되거나 교체된 기간 동안 수집되지 않은 주문은 아래 방법으로 수동으로 가져와야 합니다.
</Callout>
