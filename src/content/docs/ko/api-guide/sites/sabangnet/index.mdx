## API 연동에 관해

### 사방넷 API 연동 관련 참고사항

사방넷은 스마트십처럼 여러 쇼핑몰의 주문을 수집하여 발주 처리 및 운송장 송신 등의 작업을 처리할 수 있는 주문 통합 관리 솔루션입니다. 스마트십과 사방넷을 연동하는 경우, 스마트십은 **사방넷과의 연동만 처리**하며, 실제 주문이 발생한 쇼핑몰과의 송신은 사방넷에서 처리합니다.

따라서 다음 작업은 사방넷에서 직접 처리하셔야 합니다.

1. 원 주문 쇼핑몰에서 사방넷으로 주문 수집 및 확정(발주 처리)
2. 사방넷에서 원 주문 쇼핑몰로 운송장 번호 송신

<Callout type="info" title="연동 구조">
스마트십은 사방넷에 운송장 번호를 등록할 수 있으나, 이를 원 주문 쇼핑몰까지 전송하려면 사방넷에서 직접 처리해야 합니다. 이는 사방넷의 API 정책에 따른 것입니다.
</Callout>

## 사전 준비 — 사방넷 API 키 발급

사방넷 서비스에 대한 API 연결 설정을 위해서는 다음의 정보가 필요합니다. 스마트십에서 API 연결을 설정하기 전에 사방넷 관리도구에서 확인 및 발급을 해주세요.

| 항목 | 설명 |
|------|------|
| **샵 ID** | 사방넷에 로그인하는 판매자 ID |
| **인증키** | API 키 발급 요청 시 생성되는 고유 키 |

### API 키 발급 절차

1. [사방넷](https://www.sabangnet.co.kr/)에 로그인합니다.
2. **마이페이지 > 서비스 관리 > 연동키 관리** 메뉴로 이동합니다.
3. 해당 화면에서 연동키를 확인합니다. 스마트십 연결 시 이 값이 필요합니다.

<Callout type="info">
사방넷의 연동키는 기본적으로 유료 부가서비스입니다. API 이용 금액에 대한 문의는 사방넷으로 직접 문의해주세요.
</Callout>

## 스마트십에서 API 연결 추가

### 신규 API 연결 추가

메뉴 **주문 > 쇼핑몰 연동 > 쇼핑몰 API 설정**으로 이동합니다.

1. 상단의 **[신규등록]** 버튼을 클릭합니다.
2. 연결할 쇼핑사이트를 선택합니다. 이름 일부를 타이핑하면 빠르게 검색할 수 있습니다.
3. 목록에서 **사방넷**을 선택합니다.

### 연결 정보 입력

| 항목 | 입력 내용 |
|------|-----------|
| **쇼핑몰 샵 ID** | 사방넷 로그인에 사용하는 판매자 ID |
| **API Key** | 사방넷에서 발급받은 연동키 |

<Callout type="info">
● 표시가 있는 필드는 필수 입력 항목입니다.
</Callout>

### 연결 테스트에 실패하는 경우

연결 테스트에 실패하는 경우 다음 사항을 확인해주세요.

1. 샵 ID와 인증키(API Key)가 각각 올바른 필드에 저장되었는지 확인합니다.
2. **[등록하기]** 혹은 **[수정하기]** 버튼을 클릭하여 변경 사항을 시스템에 적용했는지 확인합니다.
3. 사방넷의 API 서버 일시적 장애나 네트워크 문제로 연결에 실패할 수 있습니다. 잠시 후 다시 시도해보세요.

위 사항들을 확인하였음에도 접속에 실패하는 경우 `apihelp@tracxlogis.com` 으로 다음 내용을 첨부하여 문의해주세요.

- 판매자 로그인 이메일 혹은 아이디
- 연결하려는 사이트 및 샵 ID
- 스마트십의 오류 화면 캡쳐
- 사방넷 API 키 발급 시 입력한 정보 화면 캡쳐

## API 설정 및 연동 주문

API 연동 동작(자동 주문 수집, 자동 주문 확인, 풀필먼트 출하지시 등)을 조절하는 옵션은 **[쇼핑몰 API 설정 > 주문 처리 설정]** 에서 변경할 수 있습니다.

<Callout type="tip">
옵션별 상세 설명은 [API 연동 시 주문 처리 옵션 가이드](/ko/api-guide/order-options/)를 참고하세요. 사방넷의 채널별 추가 옵션도 해당 페이지에서 확인할 수 있습니다.
</Callout>

### 수집 가능한 주문 상태

사방넷에서는 **신규 주문** 및 **주문 확인** 상태의 주문을 수집합니다. 취소된 주문은 수집 대상에서 제외됩니다.

<Callout type="info">
기본적으로 자동 주문 수집은 작동을 시작하는 당일부터의 주문을 수집합니다. 인증 키 만료 등의 사유로 자동 주문이 중지된 후 재설정한 경우, 주문이 수집되지 않은 기간의 주문은 반드시 수동으로 API 주문 연동을 실행해야 수집됩니다.
</Callout>

### 판매 사이트 상품에 SKU no. 등록

사방넷 **상품관리 > 사방넷상품조회수정** 화면에서 상품별로 코드를 입력합니다.

| 상품 유형 | 입력 항목 | 입력 값 |
|-----------|-----------|---------|
| 단일 상품 (옵션 없음) | 자체상품코드 | 스마트십 SKU no. 또는 판매자 정의코드 |
| 선택 상품 (옵션 있음) | 옵션별칭 | 각 옵션의 SKU no. 또는 판매자 정의코드 |

<Callout type="info">
판매채널 상품과 SKU를 연결하는 자세한 방법은 [API 연결 후 판매사이트 설정 방법](/ko/api-guide/site-settings/)을 참고하세요.
</Callout>

## API 키 관리

일반적으로 쇼핑 플랫폼이 발급하는 API 키에는 만료일자가 있습니다. 키가 만료되면 API 호출 권한이 사라져 주문 수집이나 송장 등록 등 정보 연동을 정상적으로 수행할 수 없습니다.

- API 키의 만료일자를 별도로 관리해주세요.
- 키가 만료되기 전에 사방넷에서 API 키를 갱신 또는 재발급 받아야 합니다.
- API 키를 갱신/재발급 받은 경우 일반적으로 기존 키는 즉시 만료되므로, 스마트십의 쇼핑사이트 API 설정에서도 새 키로 수정해야 합니다.

<Callout type="warning">
API 키가 만료되면 자동 주문 수집이 중단됩니다. 설정 정보를 수정한 후에는 수집이 중단된 기간의 주문을 수동으로 API 연동해주셔야 합니다.
</Callout>
