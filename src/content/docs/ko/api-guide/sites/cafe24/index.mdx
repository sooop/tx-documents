## API 연동에 관해

<ApiSiteIntro lang="ko" />

<Callout type="warning" title="Cafe24 주문처리 과정 변경 안내 (2026년 1월 26일부터)">
Cafe24 Admin API는 1초/1분당 API 호출 횟수가 매우 엄격하게 제한됩니다. 발송 처리 시 오류 및 지연을 개선하기 위해 주문 처리 프로세스가 변경되어, 스마트십에서 주문 확인(발주 확인) 시 채널 주문 상태가 다음과 같이 변경됩니다.

- 변경 이전: **배송 대기**
- 변경 후: **배송 준비중**

유의사항:

- 변경 이전에 확인 처리된 주문은 변경 후 발송 처리(배송 중 처리)에 실패할 수 있습니다. 실패 시 채널 주문의 상태를 **배송 준비중**으로 변경해주세요.
- **배송 준비중** 상태 주문의 자동 취소(판매자 승인 없이 취소)는 쇼핑몰 설정을 따릅니다. 발주 확인 후 주문이 자동 취소되지 않게 하려면, **쇼핑몰 설정 > 주문 후 설정 > 취소 요청 자동 승인** 설정에서 **배송 준비 중** 상태의 체크를 해제해주세요.
</Callout>

## 사전 준비 — Cafe24 개발자 센터 가입

Cafe24 API를 사용하려면 Cafe24 개발자 센터에 가입하여 자체 앱을 등록하고 API 인증키를 발급받아야 합니다. 개발자 센터는 쇼핑몰 관리자 계정과 별개로 회원 가입이 필요합니다.

1. [Cafe24 개발자 센터](https://developers.cafe24.com)에 접속합니다.
2. 우측 상단 **[가입]** 링크를 클릭합니다.
3. **일반회원**을 선택하고 본인 인증 후 회원 가입을 완료합니다.

## 앱 등록하기

### 신규 앱 등록

<AnnotatedImage src="cafe24-app-list" section="api-guide" alt="Cafe24 개발자 센터 App 관리 화면" caption="앱 목록에서 ADD PRODUCT를 클릭하여 신규 앱 등록" />

1. Cafe24 개발자 센터에 로그인한 후 메뉴 **Products > Apps**를 선택하여 앱 관리 화면으로 이동합니다.
2. 앱 목록 화면에서 **[ADD PRODUCT]** 를 클릭하여 새로운 앱을 등록합니다.

<AnnotatedImage src="cafe24-app-new" section="api-guide" alt="App 등록 대화 상자" caption="App 유형은 Web, 관리 상품명은 쇼핑몰 이름으로 지정" />

| 항목 | 입력 내용 |
|------|-----------|
| **App 유형** | Web |
| **관리 상품명** | cafe24 쇼핑몰 이름 |

**[저장]** 버튼을 클릭하면 앱이 등록되고 앱 상세 정보 설정 화면으로 이동합니다.

### 앱 정보 설정하기

다음 순서에 따라 신규 앱 정보를 정확하게 설정해주세요.

**App URL 입력**

아래 URL을 복사하여 입력합니다.

```
https://www.tracxlogis.com/Member/RegisterFromCafe24
```

**Redirect URI(s)**

아래 URL을 모두 추가합니다.

```
https://www.tracxlogis.com/Member/RegisterFromCafe24
https://www.tracxlogis.com/Member/RegisterFromChannels
https://www.tracxlogis.com/Member/RegisterFromChannelsV2
https://www2.tracxlogis.com/Member/RegisterFromCafe24
https://www2.tracxlogis.com/Member/RegisterFromChannelsV2
```

**권한 설정**

<AnnotatedImage src="cafe24-app-permissions" section="api-guide" alt="Cafe24 앱 권한 선택 화면" caption="각 권한을 Read + Write로 설정" />

아래 권한을 모두 추가하고, 각각 **Read + Write** 로 설정합니다.

| 분류 | 권한 |
|------|------|
| **앱(Application)** | Read + Write |
| **상점(Store)** | Read + Write |
| **상품(Product)** | Read + Write |
| **주문(Order)** | Read + Write |
| **배송(Shipping)** | Read + Write |

<Callout type="tip">
권한을 추가하려면 권한 선택 목록 상단에서 추가할 권한을 선택한 후 **[추가]** 버튼을 클릭합니다. 권한 추가 후 권한 종류를 **Read + Write** 로 변경하세요.
</Callout>

### 테스트 실행 및 계정 연결

1. 권한 설정까지 완료한 후 하단 **[저장]** 버튼을 클릭하여 설정을 저장합니다.
2. 저장 후 **[테스트 실행]** 버튼을 클릭합니다.
3. 테스트 실행 시 본인의 Cafe24 쇼핑몰 **ID**를 입력합니다.
4. 쇼핑몰 관리자 로그인 화면에서 로그인하여 해당 쇼핑몰의 관리자임을 인증합니다.
5. 인증이 완료되면 스마트십 계정 연결 화면으로 이동합니다.

<Callout type="info" title="쇼핑몰 ID란?">
쇼핑몰 ID는 쇼핑몰의 `cafe24.com` 도메인 주소의 앞부분입니다. 판매자 회원 ID가 아닙니다.

- `myshopping.cafe24.com` 으로 접속한다면 `myshopping` 부분이 쇼핑몰 ID입니다.
- 커스텀 도메인을 사용하더라도 `cafe24.com` 으로 끝나는 주소의 앞부분을 입력해주세요. 커스텀 도메인은 쇼핑몰 ID가 아닙니다.
</Callout>

### 스마트십 계정 연결

<AnnotatedImage src="cafe24-account-connect" section="api-guide" alt="Cafe24 - 스마트십 계정 연결 화면" caption="기존 계정에 연결 / 직접 입력 선택 후 스마트십 아이디 입력" />

1. **연동 방법**에서 **기존 계정에 연결**을 선택합니다.
2. 계정 선택을 **직접 입력**으로 지정합니다.
3. 스마트십 아이디(로그인 이메일)와 비밀번호를 입력하여 인증합니다.
4. 하단 **[완료]** 를 클릭하여 계정 연결을 시작합니다.
5. 연결 마법사가 실행되면서 자동으로 Cafe24 API 연동 정보가 설정됩니다. 여러 개의 몰이 설정되어 있다면 각 몰별로 API 설정이 자동 세팅됩니다.
6. 설정이 완료되면 주문 관리 화면으로 이동하여 오늘 자 신규 주문을 수집하기 시작합니다.

## 배송사 추가 설정

스마트십에 등록된 주문에 대해 1) 스마트십 내 라벨 출력 기능 사용 시, 또는 2) DPC 센터로 보낸 화물의 입고(풀필먼트 출고) 시점에 송장 번호가 API 연동을 통해 Cafe24 EC 어드민에 등록됩니다. 이 때 **송장번호를 발번한 배송사가 Cafe24 EC 어드민에 등록되어 있어야 합니다.**

<AnnotatedImage src="cafe24-shipping-company" section="api-guide" alt="Cafe24 배송업체 등록 화면" caption="쇼핑몰 관리자 > 쇼핑몰 설정 > 배송 설정 > 배송업체 관리" />

경로: **Cafe24 쇼핑몰 관리자 > 쇼핑몰 설정 > 배송 설정 > 배송업체 관리**

국내·해외 배송 시 출고되는 현지 배송사의 경우에도 동일한 방식으로 운영하는 모든 샵에 배송업체를 추가해주세요.

- 쇼핑몰 운영 정책에 따른 기본 배송비를 입력하여 배송사를 사용할 수 있도록 설정합니다.
- 배송사 등록 시에는 이름을 직접 입력하는 대신, 설정 화면의 배송사 목록에서 선택해주세요.

<Callout type="warning">
배송 업체를 등록하지 않으면 Cafe24로 송장 업데이트가 불가능합니다.
</Callout>

## API 설정 및 연동 주문

API 연동 동작(자동 주문 수집, 자동 주문 확인, 풀필먼트 출하지시 등)을 조절하는 옵션은 **[쇼핑몰 API 설정 > 주문 처리 설정]** 에서 변경할 수 있습니다.

<Callout type="tip">
옵션별 상세 설명은 [API 연동 시 주문 처리 옵션 가이드](/ko/api-guide/order-options/)를 참고하세요. Cafe24의 채널별 추가 옵션(예상 배송일자 업데이트, 재고 동기화 설정 등)도 해당 페이지에서 확인할 수 있습니다.
</Callout>

### 수집 가능한 주문 상태

Cafe24에서는 결제가 완료되었고 아직 배송 중 상태가 아닌 아래 상태의 주문을 수집합니다.

- **배송 준비 중** 상태 주문
- **배송 대기** 상태 주문

취소된 주문은 수집 대상에서 제외됩니다.

<Callout type="info">
기본적으로 자동 주문 수집은 작동을 시작하는 당일부터의 주문을 수집합니다. 인증 키 만료 등의 사유로 자동 주문이 중지된 후 재설정한 경우, 주문이 수집되지 않은 기간의 주문은 반드시 수동으로 API 주문 연동을 실행해야 수집됩니다.
</Callout>

### 판매 사이트 상품에 SKU no. 등록

<AnnotatedImage src="cafe24-product-code" section="api-guide" alt="Cafe24 상품목록 자체 품목코드 입력" caption="쇼핑몰 관리자 > 상품 > 상품목록에서 자체 상품코드/자체 품목코드에 SKU no. 입력" />

Cafe24 쇼핑몰 관리자 **상품 > 상품목록** 에서 상품별로 코드를 입력합니다. SKU와 매칭할 수 있는 상품 정보는 **자체 상품코드** 또는 옵션의 **자체 품목코드** 에 기입합니다.

| 상품 유형 | 입력 항목 | 입력 값 |
|-----------|-----------|---------|
| 단일 상품 (옵션 없음) | 자체 상품코드 | 스마트십 SKU no. 또는 판매자 정의코드 |
| 선택 상품 (옵션 있음) | 각 옵션의 자체 품목코드 | 각 옵션의 SKU no. 또는 판매자 정의코드 |

<Callout type="info">
판매채널 상품과 SKU를 연결하는 자세한 방법은 [API 연결 후 판매사이트 설정 방법](/ko/api-guide/site-settings/)을 참고하세요.
</Callout>
