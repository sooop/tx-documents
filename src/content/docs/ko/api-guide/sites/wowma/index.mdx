## 사전 준비 — Wowma! API 키 발급

Wowma!는 API 키를 직접 발급하는 방식을 사용합니다. 아래 정보를 미리 확인해두세요.

| 항목 | 설명 |
|------|------|
| **상점 코드** | Wowma! 판매자 어드민의 업체 고유 식별값 |
| **API Key** | 발급 후 90일 동안 유효한 고유 키 |

<Callout type="info">
API Key는 발급 후 **90일**이 지나면 만료됩니다. 만료 전에 갱신하지 않으면 주문 수집이 중단되므로 만료일을 별도로 관리하세요.
</Callout>

### API Key 발급 절차

<AnnotatedImage src="wowma-api-key-page" section="api-guide" alt="Wow! Manager API 이용설정 화면" caption="API利用設定 메뉴에서 약관 동의 후 키 발급" />

1. [Wow! Manager](https://wowma.jp/)에 로그인한 후 **API利用設定(API 이용설정)** 메뉴로 이동합니다.
2. 이용약관을 확인하고 동의한 후 API Key를 발급합니다.
3. **接続元IPアドレスを登録(접속 IP 주소 등록)** 항목에 아래 IP 주소를 입력합니다.

```
20.249.181.82
```

<Callout type="warning">
IP 주소를 등록하지 않으면 API 연결 테스트가 실패할 수 있습니다. 정확하게 입력했는지 반드시 확인하세요.
</Callout>

## 스마트십에서 API 연결 추가

### 신규 API 연결 추가

메뉴 **주문 > 쇼핑몰 연동 > 쇼핑몰 API 설정**으로 이동합니다.

1. 상단의 **[신규등록]** 버튼을 클릭합니다.
2. 쇼핑사이트 선택 목록에서 **Wowma!** 를 선택합니다. 이름 일부를 입력하면 빠르게 검색할 수 있습니다.

### 연결 정보 입력

<AnnotatedImage src="wowma-api-connect-form" section="api-guide" alt="스마트십 Wowma! API 연결 정보 입력 화면" caption="상점 코드, Access Key, 만료일 입력" />

| 항목 | 입력 내용 |
|------|-----------|
| **상점 코드** | Wowma! 판매자 어드민의 상점 코드(업체 고유 식별값) |
| **Access Key** | 발급받은 API Key |
| **API Key 만료일** | 발급 시 확인된 만료일자 |

## API 설정 및 연동 주문

API 연동 동작(자동 주문 수집, 자동 주문 확인, 풀필먼트 출하지시 등)을 조절하는 옵션은 **[쇼핑몰 API 설정 > 주문 처리 설정]** 에서 변경할 수 있습니다.

<Callout type="tip">
옵션별 상세 설명은 [API 연동 시 주문 처리 옵션 가이드](/ko/api-guide/order-options/)를 참고하세요.
</Callout>

### 수집 가능한 주문 상태

Wowma!에서는 **新規受付(신규주문)** 상태의 주문만 수집됩니다.

## 풀필먼트 상품 연결

풀필먼트 서비스 이용 시, 판매사이트 상품과 스마트십 SKU를 연결해야 센터에서 정확한 상품을 출고할 수 있습니다.

### 판매 사이트 상품에 SKU no. 등록

Wowma! 판매자 어드민 **商品・画像・デザイン(상품·이미지·디자인) > 新規商品登録(신규상품등록)** 에서 상품별로 코드를 입력합니다.

| 상품 유형 | 입력 항목 | 입력 값 |
|-----------|-----------|---------|
| 단일 상품 (옵션 없음) | 管理用ID | 스마트십 SKU no. 또는 판매자 정의코드 |
| 선택 상품 (옵션 있음) | 옵션별 관리 코드 | 각 옵션의 판매자 정의코드 |

<Callout type="info">
판매채널 상품과 SKU를 연결하는 자세한 방법은 [API 연결 후 판매사이트 설정 방법](/ko/api-guide/site-settings/)을 참고하세요.
</Callout>

## API 키 관리

Wowma! API Key는 발급 후 **90일** 동안만 유효합니다. 만료되기 전에 Wow! Manager에서 동일한 절차로 키를 재발급하고, 스마트십 API 설정에서 새 키로 수정해야 합니다.

<Callout type="warning">
API Key가 만료되면 자동 주문 수집이 중단됩니다. 중단된 기간의 주문은 아래 방법으로 수동으로 가져와야 합니다.
</Callout>
